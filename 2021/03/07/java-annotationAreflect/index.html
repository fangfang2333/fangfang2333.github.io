Java-注解与反射

注解

注解的定义

注解(Annontation):

  • 从jdk5.0引入的新技术
  • 注解可以被编译器读取
  • 注解有检查的约束

定义一个注解

元注解

  • 通过反射机制实现对元数据的访问呢
  • 元注解有四个：用于自定义注解，比如我需要创建一个自己的注解
  • (使用默认ctrl点进去，再点里面的值可以看见所有选项)
  • @Target ：注解可用于哪里（如类还是接口还是对象）
  • @Retention：在什么界别保存信息，描述生命周期，一般开发自定义使用RUNTIME，运行时
  • @Documented：将注解包含到javadoc中
  • @Inherited： 可以继承父类中的该注解

@Target:表示注解可以用在哪些地方

@Retention：运行界别，表示注解在什么地方还有效

  • 有源码时有效SOURCE，类创建时有效CLASS，运行时有效RUNTIME
  • 自定义的类默认写RUNTIME
  • RUANTIME > CLASS > SOURCE：从底层源码到编译，到目标程序

隐藏属性：当只有一个参数时可以在定义时使用value名，这样在赋值时就不需要指定名字

反射 Reflection

静态语言与动态语言

动态语言：

运行时可以改变结构的语言，运行时代吗可以根据某些条件改变自身结构

主要的动态语言有：Object-C、C#、JavaScript、PHP、Python

静态语言：

运行时结构不可变。如Java、C、C++

Java 不是动态语言，称为“准动态语言”。通过反射机制获得类似动态语言的特性

反射的特点

  • 允许程序执行期间借助Reflection API取得任何类的内部信息
  • 缺陷：损失一些安全性。是一种解释操作对性能有影响，

反射提供的功能

  • 运行时判断任意对象所属类
  • 运行时构造任意一个类的对象
  • 运行时判断任意一个类所具有的成员变量和方法
  • 运行时处理注解
  • 生成动态代理
  • …

一个类在内存中只有一个class对象，类被加载后，类的整个结构都会被封装在class对象中

Class 类

  • 只能通过系统建立，本身也是一个类
  • 加载类在JVM中指挥有Class实例
  • 通过Class 可以完整地得到一个类中的所有被加载的结构
  • Class 类是Reflection 的根源，针对任何动态加载、运行的类，唯有先获得相应的Class对象

通过反射获取类

拥有Class 对象的类型

  • Class：外部类，成员，局部内部
  • interface：接口，enum：枚举，annotation：注解,@interface，primitive type：基本数据类型，void

内存分析

Java内存分为堆、栈、方法区

  • 堆：存放new对象和数组

    • 可被所有线程共享，不会存放别的对象引用

  • 栈：存放基本变量类型（包含具体数值）

    • 引用对象的变量（存放引用在堆里面的具体地址）

  • 方法区：可以被所有线程共享

  • 包含了所有的class和static变量

类的加载

  • 加载：将class内容加载进内存

    • 将静态数据转换为方法区运行时的结构数据
    • 生成一个代表此类的java.lang.Class对象（通过反射）

  • 链接：将java的二进制合并到JVM的运行环境中的一个过程

    • 验证：确保是否符合规范
    • 准备：正式为变量（static）分配内存，并设置默认初始值阶段，内存在方法区进行分配
    • 解析：虚拟机常量池符号引用（常量名）替换为直接引用（地址）的过程

  • 初始化：执行构造器()方法的过程

类的引用

引用会分为主动引用和被动引用：

  • 主动引用会发生类的初始化，被动引用不会发生类的初始化

  • 主动引用产生：

    • 虚拟机启动，初始化main所在的类
    • new 一个新的对象
    • 调用静态成员和静态方法（除final）
    • 通过反射对类进行调用(java.lang.reflect)
    • 初始类时：若父类未被初始化，则初始化它的父类(类的执行过程)

  • 类的被动引用：

    • 访问静态域时，真正声明此域的累才会被初始化：当通过子类引用父类静态变量，不会导致子类初始化
    • 通过数组定义类引用，不会触发此类初始化
    • 引用常量不会触发此类初始化（常量在链接时就存入调用类的常量池中了）

类加载器

  • 类加载器：将静态数据转换为方法区的运行时结构数据，在堆中生成一个代表这个类的java.lang.Class对象
  • 类缓存：维持加载缓存。缓存是为了提高效率

类加载器作用：把类装载进内存

  • 引导加载器：C++ 编写。负责JAVA平台，JVM自带加载器，负责加载Java核心库（如rt.jar），就是根目录下的文件。无法直接获取该加载器
  • 扩展类加载器：负责/jre/lib/ext目录下的jar包，加载进工作库
  • 系统类加载器：（APP 或 System 加载器）负责java -classpath项目内家暴加载进内存
